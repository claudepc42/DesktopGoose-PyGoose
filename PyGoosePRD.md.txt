# PyGoose — Product Requirements Document
**Version:** 1.0  
**Status:** Handoff to Claude Code  
**Source:** Reverse-engineered from decompiled Desktop Goose v0.31 (arkangel-dev/desktop-goose-source)

---

## 1. Project Overview

PyGoose is a cross-platform, open-source reimplementation of samperson's Desktop Goose. It places a procedurally-animated goose on top of all windows on the user's desktop. The goose wanders autonomously, steals the cursor, tracks mud, and drags fake windows onto the screen. It supports user-customizable memes, notepad messages, and a plugin-based modding API.

The original is closed-source, Windows-only, and unmaintained. This project recreates full feature parity and adds cross-platform support and a cleaner mod API.

|----------|--------|-------|
| Windows 10/11 | Required | Primary target. Click-through overlay via Qt |

## 3. Technology Stack

| Component | Choice | Rationale |
|-----------|--------|-----------|
| Language | Python 3.11+ | Cross-platform, readable, easy modding |
| GUI / overlay | PyQt6 | Native transparent always-on-top windows, Qt painter for drawing |
| Cursor control | `pyautogui` + platform ctypes | Cross-platform cursor position/move |
| Sound | `pygame.mixer` | Cross-platform MP3/WAV, simple API |
| Config | INI via `configparser` | Match original format, user-editable |
| Packaging | PyInstaller | Single-folder distribution per platform |
| Mod API | Python `importlib` plugins | Drop-in `.py` files, cleaner than DLL injection |

## 5. Engine Layer (`engine/`)

### 5.1 `vector2.py` — Vector2

Exact port of the original `SamEngine.Vector2` struct. All operations must match precisely since the rig math depends on them.

```python
class Vector2:
    x: float
    y: float
    zero = Vector2(0.0, 0.0)

    # Operators: +, -, unary -, * (vec*vec), * (vec*float), / (vec/float)
    # Static methods:
    @staticmethod
    def get_from_angle_degrees(angle: float) -> Vector2:
        # angle * 0.0174532924  (deg2rad constant)
        return Vector2(cos(angle * 0.0174532924), sin(angle * 0.0174532924))

    @staticmethod
    def distance(a, b) -> float: ...

    @staticmethod
    def lerp(a, b, p) -> Vector2: ...

    @staticmethod
    def dot(a, b) -> float: ...

    @staticmethod
    def normalize(a) -> Vector2:
        # Guard: if x==0 and y==0, return Vector2.zero

    @staticmethod
    def magnitude(a) -> float: ...
```

### 5.2 `math_utils.py` — SamMath

```python
DEG2RAD = 0.0174532924
RAD2DEG = 57.2957764

def random_range(min: float, max: float) -> float:
    return min + random.random() * (max - min)

def lerp(a: float, b: float, p: float) -> float:
    return a * (1.0 - p) + b * p

def clamp(a: float, min_val: float, max_val: float) -> float:
    return min(max(a, min_val), max_val)
```

Single shared `random.Random()` instance throughout the app.

### 5.3 `easings.py` — Easing Functions

Implement all of the following exactly as in the original `Easings.cs`. Only `CubicEaseInOut` and `ExponentialEaseOut` are used by the goose, but implement the full set for mod authors.

Used by goose code:
- `cubic_ease_in_out(p)` — foot movement animation
- `exponential_ease_out(p)` — ESC quit progress bar

Full list to implement:

`sine_ease_in/out/in_out`, `circular_ease_in/out/in_out`,
`exponential_ease_in/out/in_out`, `elastic_ease_in/out/in_out`,
`back_ease_in/out/in_out`, `bounce_ease_in/out/in_out`

### 5.4 `deck.py` — Shuffle Deck

Fisher-Yates shuffle-on-exhaust deck. Exact behavior match is required because the task weighting depends on it.

```python
class Deck:
    def __init__(self, length: int):
        self.indices = list(range(length))
        self._i = 0
        self.reshuffle()

    def reshuffle(self):
        # Fisher-Yates: for i in range(length): swap indices[i] with indices[randint(0, i)]
        # NOTE: original uses random_range(0, i) which can return i itself — preserve this

    def next(self) -> int:
        result = self.indices[self._i]
        self._i += 1
        if self._i >= len(self.indices):
            self.reshuffle()
            self._i = 0
        return result
```

### 5.5 `time_keeper.py` — Time

```python
TARGET_FRAMERATE = 120          # Hz
DELTA_TIME = 1.0 / 120.0       # 0.008333334 seconds — FIXED, not measured

# time: float — seconds since app start, updated once per frame via time.perf_counter()
# delta_time is ALWAYS 0.008333334 regardless of actual frame timing
# This matches original exactly — the original uses a fixed delta
```

The game loop sleeps to target 120fps (`time.sleep(max(0, frame_deadline - time.perf_counter()))`). Delta time is always the fixed constant, not the measured frame time. This is how the original works and affects all physics values.

## 7. Overlay Window (`overlay.py`)

The overlay is a single frameless, transparent, always-on-top, click-through window covering the entire primary monitor.

### 7.1 Qt window flags (all platforms)

```python
flags = (
    Qt.WindowType.FramelessWindowHint |
    Qt.WindowType.WindowStaysOnTopHint |
    Qt.WindowType.Tool |              # hides from taskbar
    Qt.WindowType.NoDropShadowWindowHint
)
setAttribute(Qt.WidgetAttribute.WA_TranslucentBackground, True)
setAttribute(Qt.WidgetAttribute.WA_TransparentForMouseEvents, True)  # click-through
```

### 7.2 Platform-specific click-through

**Windows:** Additionally call `SetWindowLong` with `WS_EX_LAYERED | WS_EX_TRANSPARENT` via ctypes after the window is shown. This is required — Qt's `WA_TransparentForMouseEvents` alone is insufficient on Windows for all click-through scenarios.

```python
import ctypes
GWL_EXSTYLE = -20
WS_EX_LAYERED = 0x00080000
WS_EX_TRANSPARENT = 0x00000020

hwnd = int(window.winId())
style = ctypes.windll.user32.GetWindowLongW(hwnd, GWL_EXSTYLE)
ctypes.windll.user32.SetWindowLongW(hwnd, GWL_EXSTYLE, style | WS_EX_LAYERED | WS_EX_TRANSPARENT)
```

**macOS:** `WA_TransparentForMouseEvents` is sufficient. Accessibility permission is required for cursor manipulation — detect and prompt on first use.

**Linux/X11:** `WA_TransparentForMouseEvents` is sufficient. No extra steps.

### 7.3 Window sizing

Cover `QScreen.primaryScreen().geometry()` exactly. Do not use `availableGeometry()` (that excludes taskbar). The goose should be able to walk behind/under the taskbar visually — the taskbar will cover it naturally since it's a separate OS window.

### 7.4 Render loop

```
Application.Idle equivalent:
- QTimer with interval=0 (fires as fast as possible)
- Each tick: time_keeper.tick(), game.update(), overlay.update() (calls repaint())
- paintEvent(): game.render(painter)
- After render, sleep remainder of frame budget to target 120fps
```

Use `QPainter` with `RenderHint.Antialiasing` enabled.

## 8. Goose Rendering — The Rig

The goose is **100% procedurally drawn** using lines and ellipses. There are no sprite images. All shapes use round line caps.

### 8.1 Rig dataclass (`rig.py`)

```python
@dataclass
class Rig:
    neck_lerp_percent: float = 0.0   # 0.0=relaxed, 1.0=extended/running
    underbody_center: Vector2 = Vector2.zero
    body_center: Vector2 = Vector2.zero
    neck_center: Vector2 = Vector2.zero
    neck_base: Vector2 = Vector2.zero
    neck_head_point: Vector2 = Vector2.zero
    head1_end_point: Vector2 = Vector2.zero
    head2_end_point: Vector2 = Vector2.zero  # beak tip — used for cursor grab detection
```

### 8.2 `update_rig(position, direction, rig)` — exact math

```python
UP = Vector2(0.0, -1.0)
fwd = Vector2.get_from_angle_degrees(direction)       # forward unit vector
right = Vector2.get_from_angle_degrees(direction + 90.0)

rig.underbody_center = position + UP * 9.0
rig.body_center      = position + UP * 14.0

# neck height interpolates between 20 (relaxed) and 10 (extended)
neck_height   = int(lerp(20.0, 10.0, rig.neck_lerp_percent))
# neck forward offset interpolates between 3 (relaxed) and 16 (extended)
neck_forward  = int(lerp(3.0, 16.0, rig.neck_lerp_percent))

rig.neck_center     = position + UP * (14 + neck_height)
rig.neck_base       = rig.body_center + fwd * 15.0
rig.neck_head_point = rig.neck_base + fwd * neck_forward + UP * neck_height

rig.head1_end_point = rig.neck_head_point + fwd * 3.0 - UP * 1.0
rig.head2_end_point = rig.head1_end_point + fwd * 5.0   # beak tip
```

### 8.3 `render(painter, goose_state)` — draw order and exact sizes

Draw order (back to front):

1. **Footmarks** — brown circles, see Section 10
2. **Shadow** — hatched dark gray ellipse under body: center=(pos.x, pos.y), radii=(20, 15). Use a 2×2 dither pattern (every other pixel is DarkGray, rest transparent). Drawn with `fillEllipse`.
3. **Orange feet** — two filled orange circles, radius=4, at `l_foot_pos` and `r_foot_pos`
4. **Outline layer** (LightGray, drawn first so white overwrites):
   - Underbody line: LightGray, width=15, from `underbody_center + fwd*7` to `underbody_center - fwd*7`
   - Body line: LightGray, width=24, from `body_center + fwd*11` to `body_center - fwd*11`
   - Neck line: LightGray, width=15, from `neck_base` to `neck_head_point`
   - Head1 line: LightGray, width=17, from `neck_head_point` to `head1_end_point`
   - Head2 line: LightGray, width=12, from `head1_end_point` to `head2_end_point`
5. **White fill layer** (overwrites outline):
   - Body line: White, width=22
   - Neck line: White, width=13
   - Head1 line: White, width=15
   - Head2 line: White, width=10
6. **Beak** — Orange, width=9, from `head2_end_point` to `head2_end_point + fwd*3`
7. **Eyes** — two Black filled circles, radius=2:
   - `left_eye  = neck_head_point + UP*3 - right*b*5 + fwd*5`  where `b = Vector2(1.3, 0.4)`
   - `right_eye = neck_head_point + UP*3 + right*b*5 + fwd*5`

All lines use `Qt.PenCapStyle.RoundCap` and `Qt.PenJoinStyle.RoundJoin`.

**Custom colors:** When `UseCustomColors=True`, replace the hardcoded White body color with `GooseColorBody`, LightGray outline with `GooseColorUnderbody`, and Orange beak/feet with `GooseColorBeak`.

### 8.4 Neck lerp behavior

```python
# Each frame:
target_neck = 1.0 if (override_extend_neck or current_speed >= 200.0) else 0.0
rig.neck_lerp_percent = lerp(rig.neck_lerp_percent, target_neck, 0.075)
```

`override_extend_neck` is set to `True` during `CollectWindow.DraggingWindowBack` task.

## 9. Goose Physics and Movement

### 9.1 Speed tiers

```python
class SpeedTier(Enum):
    WALK   = "walk"
    RUN    = "run"
    CHARGE = "charge"

SPEEDS = {
    SpeedTier.WALK:   {"speed": 80.0,  "acceleration": 1300.0, "step_time": 0.2},
    SpeedTier.RUN:    {"speed": 200.0, "acceleration": 1300.0, "step_time": 0.2},
    SpeedTier.CHARGE: {"speed": 400.0, "acceleration": 2300.0, "step_time": 0.1},
}
```

### 9.2 Physics update (called every frame, `DELTA_TIME = 0.008333334`)

```python
# 1. Compute target_direction (normalized vector toward target_pos)
target_direction = Vector2.normalize(target_pos - position)

# 2. Run AI (updates target_pos, may set override_extend_neck)
run_ai()

# 3. Turn toward target (lerp angle, 25% per frame)
current_dir_vec = Vector2.get_from_angle_degrees(direction)
blended = Vector2.lerp(current_dir_vec, target_direction, 0.25)
direction = degrees(atan2(blended.y, blended.x))

# 4. Cap velocity to current_speed
if Vector2.magnitude(velocity) > current_speed:
    velocity = Vector2.normalize(velocity) * current_speed

# 5. Accelerate toward target
velocity += Vector2.normalize(target_pos - position) * current_acceleration * DELTA_TIME

# 6. Integrate position
position += velocity * DELTA_TIME

# 7. Solve feet
solve_feet()

# 8. Update neck lerp
```

### 9.3 Petting detection

```python
# In tick(), before running AI:
mouse_pos = Vector2(QCursor.pos().x(), QCursor.pos().y())
```

## 10. Footmarks

### 10.1 Ring buffer

```python
FOOT_MARK_BUFFER_SIZE = 64

@dataclass
class FootMark:
    position: Vector2 = Vector2.zero
    time: float = 0.0             # timestamp when mark was placed

foot_marks: list[FootMark] = [FootMark() for _ in range(64)]
foot_mark_index: int = 0          # wraps around
```

### 10.2 Placement

A footmark is placed whenever a foot completes a step AND `time.time < track_mud_end_time`.

```python
def add_foot_mark(pos: Vector2):
    foot_marks[foot_mark_index].time = time.time
    foot_marks[foot_mark_index].position = pos
    foot_mark_index = (foot_mark_index + 1) % FOOT_MARK_BUFFER_SIZE
```

### 10.3 Rendering

```python
FOOT_MARK_LIFETIME = 8.5    # seconds before shrink begins
FOOT_MARK_SHRINK_TIME = 1.0 # seconds to shrink from radius 3 to 0

for mark in foot_marks:
    if mark.time == 0.0:
        continue
    shrink_start = mark.time + FOOT_MARK_LIFETIME
    p = clamp((time.time - shrink_start), 0.0, 1.0) / 1.0
    radius = int(lerp(3.0, 0.0, p))
    if radius > 0:
        painter.fillEllipse(center=mark.position, radius=radius, color=SaddleBrown)
```

## 11. Foot Solver

Feet are solved using a two-foot IK system that alternates steps.

### 11.1 Foot home position

```python
FEET_DISTANCE_APART = 6
OVERSHOOT_FRACTION = 0.4
WANT_STEP_AT_DISTANCE = 5.0

def get_foot_home(right_foot: bool) -> Vector2:
    b = 1.0 if right_foot else 0.0
    side = Vector2.get_from_angle_degrees(direction + 90.0) * b
    return position + side * FEET_DISTANCE_APART
```

### 11.2 Step logic (called every frame)

Only one foot moves at a time. Steps alternate: when neither foot is moving, check left foot first.

```python
if l_foot_move_time_start < 0 and r_foot_move_time_start < 0:
    # Check if left foot needs to step
    if Vector2.distance(l_foot_pos, get_foot_home(False)) > WANT_STEP_AT_DISTANCE:
        l_foot_move_origin = l_foot_pos
        l_foot_move_dir = Vector2.normalize(get_foot_home(False) - l_foot_pos)
        l_foot_move_time_start = time.time
    # Else check right foot
    elif Vector2.distance(r_foot_pos, get_foot_home(True)) > WANT_STEP_AT_DISTANCE:
        r_foot_move_origin = r_foot_pos
        r_foot_move_dir = Vector2.normalize(get_foot_home(True) - r_foot_pos)
        r_foot_move_time_start = time.time

elif l_foot_move_time_start > 0:
    target = get_foot_home(False) + l_foot_move_dir * OVERSHOOT_FRACTION * 5.0
    elapsed = time.time - l_foot_move_time_start
    if elapsed <= step_time:
        p = elapsed / step_time
        l_foot_pos = Vector2.lerp(l_foot_move_origin, target, cubic_ease_in_out(p))
    else:
        l_foot_pos = target
        l_foot_move_time_start = -1.0
        sound.play_pat()
        if time.time < track_mud_end_time:
            add_foot_mark(l_foot_pos)

elif r_foot_move_time_start > 0:
    target = get_foot_home(True) + r_foot_move_dir * OVERSHOOT_FRACTION * 5.0
    elapsed = time.time - r_foot_move_time_start
    if elapsed <= step_time:
        p = elapsed / step_time
        r_foot_pos = Vector2.lerp(r_foot_move_origin, target, cubic_ease_in_out(p))
    else:
        r_foot_pos = target
        r_foot_move_time_start = -1.0
        sound.play_pat()
        if time.time < track_mud_end_time:
            add_foot_mark(r_foot_pos)
```

`step_time` comes from the current speed tier (0.2 for Walk/Run, 0.1 for Charge).

### 12.3 `choose_next_task()`

```python
def choose_next_task():
    # First UX sequence: first wander → TrackMud → Meme → normal rotation
    if not config.attack_randomly and time.time < config.first_wander_time_seconds + 1.0:
        set_task(Task.TRACK_MUD)
        return

    task = TASK_WEIGHTED_LIST[task_picker_deck.next()]

    # If AttackRandomly=False, skip NabMouse picks
    while not config.attack_randomly and task == Task.NAB_MOUSE:
        task = TASK_WEIGHTED_LIST[task_picker_deck.next()]

    set_task(task)
```

**First UX override:** When `AttackRandomly=False`, the very first task after the initial wander is always `TRACK_MUD`, then the next is `COLLECT_WINDOW_MEME`. The original implements this by swapping the deck's first index to point to the Meme task. Replicate this behavior: on `Init()` when `not config.attack_randomly`, swap `deck.indices[0]` to point to the index of the first `COLLECT_WINDOW_MEME` entry in the weighted list.

```python
def set_task(task, honk=True):
```

## 13. Task: Wander

```python
@dataclass
class WanderState:
    wander_start_time: float
    wander_duration: float
    pause_start_time: float = -1.0
    pause_duration: float = 0.0

# Duration constants:
MIN_PAUSE = 1.0       # seconds
MAX_PAUSE = 2.0       # seconds
GOOD_ENOUGH_DIST = 20.0

def get_random_wander_duration() -> float:
    if time.time < 1.0:
        return config.first_wander_time_seconds   # first wander uses config value
    return random_range(config.min_wandering_time_seconds, config.max_wandering_time_seconds)

def get_random_pause_duration() -> float:
    return 1.0 + random.random() * 1.0   # 1–2 seconds

def get_random_walk_time() -> float:
    return random_range(1.0, 6.0)
```

```python
def run_wander():
    # If wander duration expired, choose next task
    if time.time - task_wander.wander_start_time > task_wander.wander_duration:
        choose_next_task()
        return

    if task_wander.pause_start_time > 0.0:
        if time.time - task_wander.pause_start_time > task_wander.pause_duration:
            # Pause over — pick new target
            task_wander.pause_start_time = -1.0
            max_walk_dist = get_random_walk_time() * current_speed
            new_target = Vector2(
                random_range(0, screen_width),
                random_range(0, screen_height)
            )
            if Vector2.distance(position, new_target) > max_walk_dist:
                new_target = position + Vector2.normalize(new_target - position) * max_walk_dist
            target_pos = new_target
        else:
            velocity = Vector2.zero   # freeze during pause
    else:
        if Vector2.distance(position, target_pos) < GOOD_ENOUGH_DIST:
            # Reached target — pause
            task_wander.pause_start_time = time.time
            task_wander.pause_duration = get_random_pause_duration()
```

## 14. Task: NabMouse

Three stages: `SEEKING_MOUSE` → `DRAGGING_MOUSE_AWAY` → `DECELERATING`

```python
MOUSE_GRAB_DISTANCE = 15.0    # px, beak tip to cursor
MOUSE_SUCC_TIME = 0.06        # seconds to transition clamp rect
MOUSE_DROP_DISTANCE = 30.0    # px, when to release
GIVE_UP_TIME = 9.0            # seconds before goose gives up chasing
STRUGGLE_RANGE = Vector2(3.0, 3.0)  # max cursor clip rect size during drag
MIN_RUN_DIST_FACTOR = 2.0     # drag dest must be >= 1.2 seconds of travel at charge speed
```

```python
def run_nab_mouse():
    cursor_pos = get_cursor_position()   # platform-specific
    beak_tip = rig.head2_end_point

    if stage == Stage.SEEKING_MOUSE:
        set_speed(SpeedTier.CHARGE)
        target_pos = cursor_pos - (beak_tip - position)

        if Vector2.distance(beak_tip, cursor_pos) < MOUSE_GRAB_DISTANCE:
            # GRABBED
            original_vector_to_mouse = cursor_pos - beak_tip
            grabbed_time = time.time
            # Pick a drag destination at least 1.2 charge-seconds away
            drag_to = position
            while Vector2.distance(drag_to, position) / 400.0 < 1.2:
                drag_to = Vector2(random() * screen_w, random() * screen_h)
            target_pos = drag_to
            set_foreground_window()    # bring overlay to front
            sound.chomp()
            stage = Stage.DRAGGING_MOUSE_AWAY

        if time.time > chase_start_time + GIVE_UP_TIME:
            stage = Stage.DECELERATING

    elif stage == Stage.DRAGGING_MOUSE_AWAY:
        if Vector2.distance(position, target_pos) < MOUSE_DROP_DISTANCE:
            release_cursor_clip()
            stage = Stage.DECELERATING
        else:
            # Lerp cursor clip rect from original_vector_to_mouse → STRUGGLE_RANGE
            p = min((time.time - grabbed_time) / MOUSE_SUCC_TIME, 1.0)
            clip_vec = Vector2.lerp(original_vector_to_mouse, STRUGGLE_RANGE, p)
            # Clip rect: top-left is beak_tip adjusted for negative clip_vec components
            clip_x = beak_tip.x + clip_vec.x if clip_vec.x >= 0 else beak_tip.x + clip_vec.x
            clip_y = beak_tip.y + clip_vec.y if clip_vec.y >= 0 else beak_tip.y + clip_vec.y
            set_cursor_clip(clip_x, clip_y, abs(clip_vec.x), abs(clip_vec.y))

    elif stage == Stage.DECELERATING:
        target_pos = position + Vector2.normalize(velocity) * 5.0
        velocity -= Vector2.normalize(velocity) * current_acceleration * 2.0 * DELTA_TIME
        if Vector2.magnitude(velocity) < 80.0:
            set_task(Task.WANDER)
```

### 14.1 Platform cursor clip implementations

**Windows:** `ctypes.windll.user32.ClipCursor(ctypes.byref(RECT(left, top, right, bottom)))`. Release with `ClipCursor(None)`.

**macOS:** `Cursor.Clip` has no macOS equivalent. Simulate by moving the cursor to the beak tip position every frame using `Quartz.CGDisplayMoveCursorToPoint`. Does not confine but follows the goose convincingly.

**Linux:** Use `XGrabPointer` to confine cursor to a window region, or fall back to the macOS simulation approach with `Xlib`.

```python
def run_collect_window():
    if stage == Stage.WALKING_OFFSCREEN:
        if Vector2.distance(position, target_pos) < 5.0:
            secs_to_wait = random_range(WAIT_TIME_MIN, WAIT_TIME_MAX)
            wait_start_time = time.time
            stage = Stage.WAITING_TO_BRING_WINDOW_BACK

    elif stage == Stage.WAITING_TO_BRING_WINDOW_BACK:
        if time.time - wait_start_time > secs_to_wait:
            # Show the window in a separate thread
            threading.Thread(target=main_window.show_dialog, daemon=True).start()
            main_window.closing.connect(on_window_closed_early)

            # Pick goose destination near window's entry side
            if direction == ScreenDirection.LEFT:
                target_pos.y = lerp(position.y, screen_h / 2, random_range(0.2, 0.3))
                target_pos.x = window_width + random_range(15, 20)
            elif direction == ScreenDirection.TOP:
                target_pos.y = window_height + random_range(80, 100)
                target_pos.x = lerp(position.x, screen_w / 2, random_range(0.2, 0.3))
            elif direction == ScreenDirection.RIGHT:
                target_pos.y = lerp(position.y, screen_h / 2, random_range(0.2, 0.3))
                target_pos.x = screen_w - (window_width + random_range(20, 30))

            # Clamp target so goose stays visible
            target_pos.x = clamp(target_pos.x, window_width + 55, screen_w - window_width - 55)
            target_pos.y = clamp(target_pos.y, window_height + 80, screen_h)

            stage = Stage.DRAGGING_WINDOW_BACK

    elif stage == Stage.DRAGGING_WINDOW_BACK:
        if Vector2.distance(position, target_pos) < 5.0:
            # Done — walk away
            target_pos = position + Vector2.get_from_angle_degrees(direction + 180.0) * 40.0
            set_task(Task.WANDER)
            return

        override_extend_neck = True
        target_direction = position - target_pos   # face backward (away from window)
        # Pin window to beak
        window_pos = rig.head2_end_point - window_offset_to_beak
        main_window.move_threadsafe(int(window_pos.x), int(window_pos.y))
```

## 16. Task: TrackMud

Three stages: `DECIDE_TO_RUN` → `RUNNING_OFFSCREEN` → `RUNNING_WANDERING`

```python
TRACK_MUD_DURATION = 15.0       # seconds of mud tracking
DIR_CHANGE_INTERVAL = 100.0     # seconds (effectively never changes direction)
AMOK_DURATION = 2.0             # seconds of manic running before slowing
```

```python
def run_track_mud():
    if stage == Stage.DECIDE_TO_RUN:
        _set_target_offscreen()
        set_speed(SpeedTier.RUN)
        stage = Stage.RUNNING_OFFSCREEN

    elif stage == Stage.RUNNING_OFFSCREEN:
        if Vector2.distance(position, target_pos) < 5.0:
            target_pos = Vector2(random_range(0, screen_w), random_range(0, screen_h))
            next_dir_change_time = time.time + DIR_CHANGE_INTERVAL
            time_to_stop_running = time.time + AMOK_DURATION
            track_mud_end_time = time.time + TRACK_MUD_DURATION
            stage = Stage.RUNNING_WANDERING
            sound.play_mud_squish()

    elif stage == Stage.RUNNING_WANDERING:
        if (Vector2.distance(position, target_pos) < 5.0
                or time.time > next_dir_change_time):
            target_pos = Vector2(random_range(0, screen_w), random_range(0, screen_h))
            next_dir_change_time = time.time + DIR_CHANGE_INTERVAL

        if time.time > time_to_stop_running:
            # Slow down and return to wander
            target_pos = position + Vector2(30.0, 3.0)
            target_pos.x = clamp(target_pos.x, 55.0, screen_w - 55.0)
            target_pos.y = clamp(target_pos.y, 80.0, screen_h - 80.0)
            set_task(Task.WANDER, honk=False)
```

`track_mud_end_time` is a global float (initialized to `-1.0`). Footmarks are added only while `time.time < track_mud_end_time`.

## 19. Base Movable Window (`movable_window.py`)

Both NotepadWindow and MemeWindow inherit from this.

```python
class MovableWindow(QWidget):
    closing = pyqtSignal()    # emitted when user closes window

    def __init__(self):
        super().__init__()
        self.setWindowFlags(
            Qt.WindowType.FramelessWindowHint |   # or keep frame for notepad
            Qt.WindowType.WindowStaysOnTopHint |
            Qt.WindowType.Tool
        )

    def move_threadsafe(self, x: int, y: int):
        # Use QMetaObject.invokeMethod or a queued signal to move from game thread
        QMetaObject.invokeMethod(self, "_do_move", Qt.ConnectionType.QueuedConnection,
                                 Q_ARG(int, x), Q_ARG(int, y))

    @pyqtSlot(int, int)
    def _do_move(self, x, y):
        self.move(x, y)
        # ensure stays on top
        self.raise_()

    def closeEvent(self, event):
        self.closing.emit()
        super().closeEvent(event)
```

NotepadWindow uses standard window chrome (with title bar). MemeWindow is frameless.

## 21. Quit Mechanic

The original requires holding ESC for ~5 seconds to quit. Replicate exactly.

```python
QUIT_ALPHA_INCREMENT = 0.00216666679  # per frame while ESC held
QUIT_ALPHA_DECREMENT = 0.0166666675   # per frame while ESC released
QUIT_THRESHOLD = 0.99                 # quit when alpha exceeds this
QUIT_SHOW_THRESHOLD = 0.2            # show bar when alpha exceeds this
```

```python
def update_quit(painter, keys_pressed):
    if Qt.Key.Key_Escape in keys_pressed:
        quit_alpha += QUIT_ALPHA_INCREMENT
    else:
        quit_alpha -= QUIT_ALPHA_DECREMENT
    quit_alpha = clamp(quit_alpha, 0.0, 1.0)

    if quit_alpha > QUIT_SHOW_THRESHOLD:
        frac = (quit_alpha - 0.2) / 0.8
        # y offset: lerp(-15, 10, exponential_ease_out(frac * 2)) — slides in from top
        y = int(lerp(-15, 10, exponential_ease_out(frac * 2)))
        text = "Continue holding ESC to evict goose"
        font = QFont("Arial", 12, QFont.Weight.Bold)
        metrics = QFontMetrics(font)
        text_rect = metrics.boundingRect(text)
        w = text_rect.width() + 10
        h = text_rect.height() + 10

        # LightBlue background rect
        painter.fillRect(5, y - 5, w, h, QColor("LightBlue"))
        # LightPink progress fill (proportional to quit_alpha)
        painter.fillRect(5, y - 5, int(lerp(0, w, quit_alpha)), h, QColor("LightPink"))
        # Text
        alpha_byte = int(256 * quit_alpha)
        painter.setPen(QColor(alpha_byte, alpha_byte, alpha_byte))
        painter.setFont(font)
        painter.drawText(10, y + text_rect.height(), text)

    if quit_alpha > QUIT_THRESHOLD:
        QApplication.quit()
```

The overlay window must receive keyboard events — set `setFocusPolicy(Qt.FocusPolicy.StrongFocus)` and call `grabKeyboard()` on startup, OR poll keyboard state each frame using platform APIs.

## 24. Platform Notes for Claude Code

### Windows
- Click-through: must call `SetWindowLong` via ctypes after window show. Qt alone is not enough.
- Cursor clip: `ClipCursor` via ctypes, `RECT` struct.
- Bring to foreground: `SetForegroundWindow(hwnd)` when goose grabs cursor.
- Sound: `pygame.mixer` with MP3 support (may need `pygame` built with MP3; alternatively use `playsound` or `pydub` as fallback).

### macOS
- Accessibility permissions required for: cursor position read, cursor move. Use `pyautogui` which handles the permission check.
- Always-on-top: Qt handles this well on macOS.
- No `ClipCursor` equivalent — simulate with rapid `pyautogui.moveTo()` in the drag loop.
- App must be signed or user must approve in Security & Privacy for Accessibility.

### Linux
- Check for `DISPLAY` env var; abort with message if not set.
- `pyautogui` uses `python-xlib` under Linux — ensure it's in requirements.
- Cursor confinement: `pyautogui` move simulation or `Xlib.ext.xtest` for more reliable cursor control.
- `pygame.mixer` should work on Linux with ALSA/PulseAudio.
