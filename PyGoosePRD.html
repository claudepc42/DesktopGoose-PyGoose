01 Project Overview

PyGoose is a cross-platform, open-source reimplementation of samperson's Desktop Goose. It places a procedurally-animated goose on top of all windows on the user's desktop. The goose wanders autonomously, steals the cursor, tracks mud, and drags fake windows onto the screen. It supports user-customizable memes, notepad messages, and a plugin-based modding API.

Why rebuild it? The original is closed-source, Windows-only, and unmaintained. This project recreates full feature parity and adds cross-platform support and a cleaner mod API.

def random_range(min: float, max: float) -> float:
    return min + random.random() * (max - min)

def lerp(a: float, b: float, p: float) -> float:
    return a * (1.0 - p) + b * p

def clamp(a: float, min_val: float, max_val: float) -> float:
    return min(max(a, min_val), max_val)

5.4 Shuffle Deck

Fisher-Yates shuffle-on-exhaust deck. Exact behavior match is required because task weighting depends on it.

engine/deck.py
class Deck:
    def __init__(self, length: int):
        self.indices = list(range(length))
        self._i = 0
        self.reshuffle()

    def reshuffle(self):
        # Fisher-Yates: for i in range(length): swap indices[i] with indices[randint(0, i)]
        # NOTE: original uses random_range(0, i) which can return i itself — preserve this

    def next(self) -> int:
        result = self.indices[self._i]
        self._i += 1
        if self._i >= len(self.indices):
            self.reshuffle()
            self._i = 0
        return result

5.5 Time Keeper

engine/time_keeper.py
TARGET_FRAMERATE = 120          # Hz

# time: float — seconds since app start, updated once per frame

!Fixed delta time: Delta time is always the constant, not the measured frame time. This matches the original exactly and affects all physics values. TimeKeeper.sleep_remainder() is retained for reference but unused — pacing comes from the QTimer interval.

The simulation runs at 120 Hz (two fixed steps per 16 ms timer wake) while rendering runs at 60 Hz with dirty-rect partial repaints.

07 Overlay Window

A single frameless, transparent, always-on-top, click-through window covering the entire primary monitor.

Qt window flags (all platforms)

overlay.py
flags = (
    Qt.WindowType.FramelessWindowHint |
    Qt.WindowType.WindowStaysOnTopHint |
    Qt.WindowType.Tool |              # hides from taskbar
    Qt.WindowType.NoDropShadowWindowHint
)
setAttribute(Qt.WidgetAttribute.WA_TranslucentBackground, True)
setAttribute(Qt.WidgetAttribute.WA_TransparentForMouseEvents, True)

Platform-specific click-through

Windows

Call SetWindowLong with WS_EX_LAYERED | WS_EX_TRANSPARENT via ctypes after the window is shown. Qt's WA_TransparentForMouseEvents alone is insufficient.

macOS

After show(), call setIgnoresMouseEvents_(True) on each NSWindow via ObjC ctypes. Also call setHidesOnDeactivate_(False) so windows stay visible when another app gains focus.

Window sizing: Cover QScreen.primaryScreen().geometry() exactly — not availableGeometry(). The goose should be able to walk behind the taskbar visually.

Render loop (120Hz sim / 60Hz render)

A single QTimer(interval=16) drives the loop. Each wake calls on_tick() twice (two fixed 1/120s physics steps), then schedules a partial dirty-rect repaint.

iDirty-rect repainting: Goose.dirty_rect() returns the bounding box of the goose plus any footmarks mid-shrink. The overlay unions current + previous frame rects and calls self.update(region) instead of a full-screen repaint. When the ESC quit bar is visible, its rect is unioned in.
iIdentical-frame skipping: Goose.dirty_rect() returns None when nothing affecting drawn pixels has changed. The overlay skips the repaint entirely. During static behaviors (sit, fake-sleep, stand-still), this drops CPU from ~10% to ~2% of one core.
✓Performance result: v0.33 reduced idle CPU from ~12.5% to ~0.5%. The lever is update frequency, not region area — Qt re-blits the whole translucent layered window regardless of the invalidated region.

Physics update (per frame)

physics loop — DELTA_TIME = 0.008333334
# 1. Compute to_target
to_target = Vector2.normalize(target_pos - position)

# 2. Run AI (updates target_pos, may set override_extend_neck, _freeze_position)
run_ai()

# 3. Freak-out override (overrides target_pos/speed if active)

# 4. Turn toward target (lerp angle, 25% per frame)
current_dir_vec = Vector2.get_from_angle_degrees(direction)
blended = Vector2.lerp(current_dir_vec, to_target, 0.25)
direction = degrees(atan2(blended.y, blended.x))

# 5. If _freeze_position: zero velocity, return early
if _freeze_position:
    velocity = Vector2.zero; return

# 6. Cap velocity to current_speed
if Vector2.magnitude(velocity) > current_speed:
    velocity = Vector2.normalize(velocity) * current_speed

# 7. Accelerate toward target
velocity += Vector2.normalize(target_pos - position) * current_acceleration * DELTA_TIME

# 8. Integrate position
position += velocity * DELTA_TIME

10 Footmarks

A ring buffer of 64 FootMark records. A mark is placed whenever a foot completes a step AND time.time < track_mud_end_time.

for mark in foot_marks:

11 Foot Solver

Two-foot IK system that alternates steps. Only one foot moves at a time; when neither is moving, left foot is checked first.

foot home position
FEET_DISTANCE_APART  = 6.0
OVERSHOOT_FRACTION   = 0.4
WANT_STEP_AT_DISTANCE = 5.0

def get_foot_home(right_foot: bool) -> Vector2:
    s = rig.sit_lerp_percent
    side = Vector2.get_from_angle_degrees(direction + 90.0) * (1.0 if right_foot else 0.0)
    perp_dist  = lerp(FEET_DISTANCE_APART, 2.0, s)
    crawl_drop = lerp(0.0, 8.0, s)
    return position + side * perp_dist + Vector2(0.0, crawl_drop)

When a foot steps, it overshoots slightly toward get_foot_home() + move_dir * OVERSHOOT_FRACTION * 5.0, animated over step_time using cubic_ease_in_out. On step completion: play pat sound, add footmark if mud active.

    set_speed(CHARGE)
    target_pos = cursor_pos - (beak_tip - position)
    if distance(beak_tip, cursor_pos) < MOUSE_GRAB_DISTANCE:

15 Task: CollectWindow (Meme + Notepad)

Goose fetches a new window from offscreen and drops it on the desktop. Maximum 2 meme windows and 2 notepad windows on screen at once (independent pools).

Stage flow

full stage sequence
# If 2 of this type already on screen — eviction path:
WALKING_TO_EVICT → EVICTING_WINDOW →

# Always:
WALKING_OFFSCREEN → WAITING_TO_BRING_WINDOW_BACK → DRAGGING_WINDOW_BACK

WAIT_TIME_MIN = 2.0    # seconds waiting offscreen
WAIT_TIME_MAX = 3.5

Eviction behavior

Goose walks to the grab edge of the evict window (right edge for left-side window, left edge for right-side window), extends neck, attaches it to its beak, and walks toward the nearest screen edge (−80px past left or +80px past right). On disappearing offscreen, window is hidden and deleted.

iDisconnect the evicted window's closing signal before hiding — prevents the anger callback from firing when the goose itself removes it.

Placement position

First window of its type: wide random spread up to 300px from entry edge, full vertical range. Second window: same base + messy offset (50–130px further, ±150px perpendicular jitter). The goose is not neat.

Placed window anger

The most recently placed window is the "anger window." If the user closes it:

  • Notepad: goose attacks indefinitely (no timeout)
  • Meme: goose gets angry only within 3 seconds of placing

Both trigger set_task(Task.NAB_MOUSE). Closing any non-anger window just removes it from tracking with no reprisal.

class WatchSubState(Enum):
    STAND_STILL = "stand_still"
    WALK_SLOW   = "walk_slow"
    SIT         = "sit"          # crouches, _target_sit_lerp = 1.0
  • STAND_STILL: _freeze_position = True, standing pose
  • WALK_SLOW: walk toward cursor at WALK speed if dist > 60px, otherwise freeze
  • SIT: freeze position, _target_sit_lerp = 1.0

Head bob: Every 1.2–3.5s (except SIT), override neck extension for 0.35s. Rare honk: Every 5–12s, 30% chance. Always faces cursor by setting target_pos = position + normalize(cursor - position) * 50.0.

Petting while sitting: 70% honk, 20% wander, 10% attack mouse.

Stages: RUSHING → FOLLOWING ↔ FLEEING

constants

  • RUSHING: CHARGE toward preferred distance, then switch to FOLLOWING
  • FOLLOWING: Maintain preferred distance. Freeze within ±35px deadband. Flee if cursor < 45px. Occasional honk march (rapid honking for 2.5s).
  • FLEEING: Run 180px away from cursor at RUN speed for 1.5s, return to FOLLOWING

Boredom: After 15–30s, 5% chance to snap into NAB_MOUSE, otherwise wander.

  • DRAGGING: Same as NabMouse DRAGGING_MOUSE_AWAY. Continue honking.
  • DECELERATING: Same as NabMouse DECELERATING.

20 Task: Sleep

Goose walks to a screen corner, circles in a shrinking spiral, settles, and sleeps. 15% chance of fake sleep.

constants
SLEEP_CIRCLE_RADIUS   = 88.0
SLEEP_SETTLE_DURATION = 2.2
SLEEP_MIN_DURATION    = 90.0    # seconds
SLEEP_MAX_DURATION    = 480.0   # 8 minutes
SLEEP_CORNER_MARGIN   = 165.0   # px inset from screen edge

# Stages: WALKING_TO_CORNER → CIRCLING → SETTLING → SLEEPING

CIRCLING: SNEAK speed. Target advances along a shrinking spiral (1.5 revolutions, constant arc speed 40px/s). Transitions to SETTLING at spiral_t >= 0.6.

SETTLING: Over 2.2s, lerp _target_sit_lerp and _target_neck_tuck from 0→1. Then roll 15% for fake sleep.

Fake sleep behavior

  • No sleep bubbles rendered
  • Every 5–15s: open one random eye for 0.5–2.5s, then close
  • While eye open + cursor within 150px: start spotted sequence
    • 0–0.75s: hold still (one eye open)
    • 0.75–1.5s: both eyes open (peek_eye = 3) + show exclamation
    • >1.5s: trigger freak-out (§9 physics override)

Real sleep behavior

  • Sleep bubbles rendered (rig.show_sleep_bubbles = True)
  • Clicking the goose wakes it (honk, transition to WANDER)

21 Task: PeekBack

After a fake-sleep freak-out, the goose returns to the nearest screen edge, peeks back in cautiously, sweeps its gaze, then walks back onto the screen.

constants
PEEK_INSET = 14.0   # px from screen edge for the peek position

# Stages: peeking_in → looking → walking_in → pausing

Setup: Find nearest edge. Snap rig immediately to full crawl (sit_lerp = 1.0, neck_tuck = 1.0). Set sweep_deg to random 45–150°.

looking stage: Sweep gaze left/right using a sine wave over 8.8s with smoothstep ease-in/out envelope over first/last 18% of duration, linear exit blend toward enter_pos direction in final 18%.

walking_in: SNEAK speed. Gradually stand up: _target_sit_lerp and _target_neck_tuck lerp from 1→0 proportional to distance remaining.

pausing: Freeze 0.5–1.5s, then WANDER.

24 Base Movable Window

Both NotepadWindow and MemeWindow inherit from MovableWindow.

movable_window.py
class MovableWindow(QWidget):
    closing = pyqtSignal()    # emitted when user closes window

    def move_threadsafe(self, x: int, y: int):
        QMetaObject.invokeMethod(self, "_do_move", QueuedConnection,
                                 Q_ARG(int, x), Q_ARG(int, y))

    @pyqtSlot(int, int)
    def _do_move(self, x, y):
        self.move(x, y)
        self.raise_()

    def showEvent(self, event):
        super().showEvent(event)
        if sys.platform == "darwin":
            from pygoose.goose.overlay import _macos_fix_hides_on_deactivate
            _macos_fix_hides_on_deactivate()

    def closeEvent(self, event):
        self.closing.emit()
        super().closeEvent(event)
imacOS showEvent: _macos_fix_hides_on_deactivate() sets setHidesOnDeactivate:False so windows stay visible when another app gains focus. It does NOT set click-through — that would make meme/note windows unclickable.

29 Startup Sequence

  1. Create QApplication

  2. Detect platform; if Wayland and no XWayland: show error dialog and exit

  3. Load config.ini (create silently with defaults if missing)

  4. Create overlay window (transparent, always-on-top, click-through)

  5. Size overlay to primary monitor via QScreen.primaryScreen().geometry()

  6. Apply platform-specific click-through (Windows: SetWindowLong; macOS: ObjC ctypes)

  7. Init sound system (unless SilenceSounds=True)

  8. Init goose: position (-20, 120), target (100, 150), task WANDER (honk=False)

  9. If EnableMods=True: discover and load mods

  10. Start game loop timer; QApplication.exec()

Path resolution

pygoose/paths.py
def resource_path(*parts) -> str:
    """Bundled read-only assets: sounds, fonts."""
    base = getattr(sys, '_MEIPASS', os.path.dirname(os.path.abspath(__file__)))
    return os.path.join(base, *parts)

def user_data_path(*parts) -> str:
    """User-editable content: config.ini, memes, notepad messages."""
    if getattr(sys, 'frozen', False):
        base = os.path.dirname(sys.executable)
    else:
        base = os.path.dirname(os.path.abspath(__file__))
    return os.path.join(base, *parts)
